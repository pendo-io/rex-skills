## Sentence case

Use sentence case for all UI elements, including:
- Buttons
- Labels
- Tooltips
- Empty state titles
- Page names
- Section headers

Exceptions:
- **Global navigation** may use title case.
- **Proper nouns** (e.g., Pages, Features) should remain capitalized.

## Punctuation

- Use periods at the end of complete sentences.
- Don't add punctuation to button labels or headings unless it's a question.
- Use common contractions (e.g., "you're," "can't") for tone.
- Avoid em dashes and slashes. Use "to," or parentheses if necessary.

## Lists

- Use numbered lists for step-by-step instructions.
- Use bullets for unordered options or clarifying information.
- Use sentence case and end each list item with a period if it's a full sentence.

## Formatting

| Element | Format example |
|---|---|
| Button | Tag Feature |
| Tooltip title | Why can't I update this setting? |
| Notification | Your settings have been updated. |
| Error message | This field is required. |
| Checkbox label | Send me updates |

## Readability tips

- Use short paragraphs and break long sentences.
- Place key information first in headings and messages.
- Support keyboard/screen reader accessibility with meaningful labels.
